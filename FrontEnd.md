# XY Assistant 前端可视化方案备忘

## 1. 背景与目标
- 需求来源：专家评审展示，需要一套“用户输入 → 大模型推理 → 结果输出”的全链路可视化 PPT。
- 核心目标：用交互式网页还原混合工作流，强调技术复杂度、推理路径和延迟指标，同时保持高端审美。
- 原项目保持不受影响，要求在独立 `visualizer/` 前端子目录内实现。

## 2. 关键需求（按时间整理）
1. **总览 → 细节的 PPT 结构**：首页为项目全景与亮点，后续幻灯片详解执行链路，左右切换、16:9 自适应铺满屏幕。
2. **视觉与动效**：白色主背景，低饱和多巴胺配色，平滑高阶动画，数据流动需有动态连线，图标/插画占比高、文字精炼。
3. **交互式链路演示**：
   - 输入任意指令（示例：“请帮我定一个明早8点的闹钟”）后，逐步展示代码中每个处理模块（规则引擎、LLM、工具调用等）。
   - 展示真实毫秒/秒级耗时，与后端实际返回一致；复杂推理要展示详细判断过程。
   - 数据流过程保持在单页内，可暂停/回放，每一步有足够停留时间供讲解。
4. **Canvas 工作流编排**：
   - 各模块以节点形式放置在画布，可自由拖拽、支持鼠标滚轮缩放及空白处拖动画布。
   - 同级并行工具需要并列展示，节点间用箭头线段标注数据传递，提示输出内容。
   - 参考 Dify 编排体验，突出模块职责（SRP）、扩展能力（OCP）。
5. **接口集成**：
   - 后端已在 `http://0.0.0.0:8000` 启动。
   - 前端需以 `POST /api/command` 调用 FastAPI，OPTIONS 405 原因是缺少对应路由，后续在 `app/main.py` / `app/routers/command.py` 增补 POST 支持与 CORS。
6. **布局自适应**：
   - 页面主体与 Canvas 均需自适应不同分辨率、放大主体内容，前端整体铺满窗口。

## 3. 当前实现概览
- **技术栈**：Vite + React 19 + TypeScript + TailwindCSS + Framer Motion + Lucide 图标，遵循 KISS/YAGNI（仅实现展示所需功能）。
- **页面结构**（`visualizer/src/App.tsx`）：
  - 幻灯片容器组件（`Slide`）支持横向切换和全屏占位。
  - **Slide 1**：`RequestSummary`、`ScenarioGallery`、`IntentAtlas` 等模块，呈现项目总览、案例选择、意图矩阵。
  - **Slide 2**：`InteractiveChain` 提供链式调用演示，含拓扑、数据流动画、实时输出面板。
- **数据模型**：
  - `src/types/trace.ts` 定义前后端共享的 trace schema。
  - `src/data/` 下 `scenarios.ts`、`intentAtlas.ts` 提供静态示例，可随时切换。
- **交互细节**：
  - `InteractiveChain.tsx` 内建节点拖拽、画布平移/缩放、节点状态动画与进度条。
  - 动画通过 Framer Motion 控制透明度/位置/描边流动；绘制箭头时展示实时数据包内容与耗时。
  - API 请求读取 `.env` 中 `VITE_API_BASE`，默认 `http://0.0.0.0:8000`，成功后将真实 trace 写入画布。

## 4. 后端调整摘要
- `app/main.py`：挂载 `CORSMiddleware`，允许来自前端开发端口（localhost/127.0.0.1/0.0.0.0:5173）的跨域 POST。
- `app/routers/command.py`：新增 `@router.post("/command")`，处理 `CommandRequest → CommandResponse`，解决 405 问题并提供前端所需的链路数据。

## 5. 设计原则落地
- **KISS**：组件按展示功能拆分，逻辑清晰，避免额外状态。
- **YAGNI**：仅构建所需的两页展示与链式演示，未引入复杂图表库或未定义需求的功能。
- **DRY**：共享类型与场景数据集中管理；动画与布局样式通过 Tailwind 主题复用。
- **SOLID**：
  - SRP：各组件只负责自身板块（如 `IntentAtlas` 聚焦意图矩阵）。
  - OCP：`InteractiveChain` 的节点/连接由数据驱动，添加新模块无需改写核心渲染逻辑。
  - DIP：前端依赖抽象 trace 结构，不耦合具体后端实现。

## 6. 已知体验优化点
- 画布滚轮缩放无惯性抑制，后续可引入平滑因子。
- 浏览器尺寸变化会重置节点位置，暂未做智能居中。
- 未覆盖移动端触控手势，需要额外手势支持。
- 长文本、极长推理链的排版尚需压力测试；需留意文本溢出与动画节奏。

## 7. 推荐后续工作
1. 引入轻量化 E2E/集成测试（Playwright/Testing Library）覆盖 API 交互基础流程。
2. 为 `InteractiveChain` 增加节奏控制（暂停/快进）与步骤说明气泡，增强展示节奏掌控。
3. 接入实时日志（SSE/WebSocket）实现逐行回放，凸显系统实时性。
4. 根据审阅反馈微调配色与动效，确保专家现场观感佳。
5. 记录可选的节点布局预设，避免全部重置导致的讲解断点。
